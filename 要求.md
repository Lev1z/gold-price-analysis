# Python 程序设计之课程设计

## 关于本设计：

- 依据教学大纲规定，本学期本课程将要求学生完成一项课程设计，该设计占总评成绩10%的权重（其中项目内容完成度占比5%，汇报表现占比5%）。本设计需要班内学生组队完成，每个队伍不超过3人，请同学们自行组队。选定队伍后，请大家在5月8日前将队伍信息填入指定的在线腾讯文档内（教师稍后微信群提供）。出场顺序将于稍后随机安排。

- 本设计需要制作汇报ppt，并在本学期第17，18周进行汇报（此为预计时间，具体视教学进度而定）。每小组的汇报时长为6分钟（严格计时，不允许超时）。每小组需选派1-3人介绍项目内容，汇报当天全体队员需要到场出席，未出席的同学将记为项目未完成。汇报后，每个小组需要在第十八周周五下午5：00前将ppt上传至 头歌系统图文作业栏（教师稍后微信群提供链接），文件名为“python大作业-组名-大作业标题”。如ppt文件过大请转换为pdf上传（每个小组上传一份即可，因为教学材料备案需要，每个小组需要起一个组名）

## 设计要求

- 本设计分为三部分内容，第一部分为编写爬虫程序爬取网页信息，本部分要求学生利用urllib，request，scrapy，bs4等package编写爬虫程序，从公开网站的页面中爬取有用的信息并保存。第二部分为数据分析与可视化实操，本部分要求学生利用NumPy，SciPy，pandas，statistics等package对第一部分爬取的信息进行数据分析及作图。第三部分为AI辅助数据处理与分析（挑战模块，可选加分）。要求在自行完成爬虫数据获取的基础上，利用AI工具或大语言模型对数据进行分析与应用（如文本分类、情感分析、关键词提取、摘要生成等），并可进一步尝试构建简单的Agent或基于爬取数据的RAG问答系统，实现对数据的智能检索与问答。需明确：爬虫程序必须由学生独立完成，AI不得替代数据采集过程。鼓励对AI结果进行验证，并与传统分析方法对比。汇报中需说明AI工具、方法及其作用，教师将根据创新性与分析深度给予加分。

- 具体示例可参考教师第九讲-2课程内容以及相关补充课件。

- 学生报告需要ppt内容（包含但不限于）：
  1. 通过分析网络源代码爬取页面信息的简单过程以及思路介绍；
  2. 对数据分析实操中获得的图表进行说明及阐述。
  3. 对 AI 辅助数据处理与分析部分进行说明，包括使用的 AI 工具、处理思路、分析结果及与传统方法的对比。
  4. 项目中遇到怎样的困难以及解决方案（鼓励学生提高项目质量，如果是很有价值的困难被顺利解决，将获得相应加分）
  
  汇报无需代码细节详细介绍。

## 注意事项：

本作业如果小组选取复现现成案例，必须在ppt内申明现有案例的来源（包括但不限于：书本、网址或来自于小组成员别的课程大作业成果）。若采取此种方式，ppt内应着重介绍小组在现有成果上做出的改进与升级。教师将根据小组的原创工作质量对作业进行打分。如使用现成案例未申报，该小组将被记为项目未完成。
