# 黄金价格智能分析平台 — 团队计划与分工

## 项目概述

构建一个完整的黄金价格分析链路：**爬虫获取金价+新闻数据 → 数据分析与可视化 → RAG 问答系统 → 深度学习预测探索**。覆盖课程三大模块要求，AI 部分争取加分。

## 团队分工

| 成员 | 年级 | 主要职责 |
|------|------|---------|
| zpp | 大二 | RAG 系统、深度学习预测、整体架构与协调 |
| A | 大一 | 金价爬虫 + 新闻爬虫 + 数据可视化 |
| B | 大一 | 数据清洗 + 周期性分析 + Streamlit 界面 + PPT |

---

## 四周时间线

### 第一周：爬虫

| 谁 | 做什么 |
|------|--------|
| A | 用 `requests` + `BeautifulSoup` 爬东方财富/新浪等公开页面的金价历史数据和新闻，存入 SQLite |
| B | 协助 A 确认 API 接口，数据入库后做去重、补空等清洗工作 |
| zpp | 确认可用接口，指导爬虫思路，帮 debug |

**阶段交付：** `data/gold.db` 里有完整的历史金价数据 + 新闻数据。

---

### 第二周：数据分析 + 可视化

| 谁 | 做什么 |
|------|--------|
| A | 用 `pandas` 做基础统计分析（均值、标准差、收益率），用 `matplotlib` + `mplfinance` 产出 K 线图、均线图、事件标注图、收益率分布图 |
| B | 做周期性分析（周内效应、月度对比），产出对应图表 |
| zpp | 搭建 RAG 系统基础框架 |

**阶段交付：** PNG 图表文件可直接用于 PPT。

---

### 第三周：AI 模块

| 谁 | 做什么 |
|------|--------|
| zpp | RAG 问答系统：ChromaDB 建向量索引 + DeepSeek API 实现检索问答；ARIMA 基线预测 + LSTM 探索 |
| B | 接上周完成的图表，用 Streamlit 搭建交互界面（价格走势 + 问答输入框 + 预测展示） |
| A | 协助验证 RAG 检索效果，补充新闻数据 |

**阶段交付：** 可运行的 RAG 问答系统 + Streamlit 界面 + 预测对比图。

---

### 第四周：整合 + PPT + 汇报

| 谁 | 做什么 |
|------|--------|
| 全员 | 代码整合、PPT 制作、汇报演练 |

**PPT 结构建议（6 分钟）：**
1. 封面/背景 — B（30s）
2. 爬虫思路 — A（1min）
3. 数据分析与可视化 — A（1.5min）
4. RAG 问答系统 — zpp（1min）
5. 价格预测探索 — zpp（1min）
6. 遇到的困难与解决方案 — 全员（30s）
7. 总结 — B（30s）

**建议讲的"困难"方向：** 反爬处理、中文字体乱码、Streamlit 自学、预测模型效果分析。

## 协作原则

- **代码统一提交**，每次 commit 写明改动内容
- 有阻塞**先自己尝试解决**，超过半天搞不定群里说
- 不需要依赖等待：爬虫不依赖可视化，Streamlit 先用占位图，RAG 不依赖界面

## 风险预案

| 风险 | 应对 |
|------|------|
| 网站反爬 | 换数据源或手动下载 CSV |
| LSTM 效果差 | 作为对比分析呈现即可，不扣分反加分 |
| 时间不够 | 优先级：爬虫 > 可视化 > RAG > Streamlit > LSTM |
