# 黄金价格智能分析平台 — 设计文档

## 项目概述

基于 Python 爬虫、数据分析与可视化、RAG 问答系统及深度学习探索的黄金价格分析课程设计项目。满足 Python 课程设计三大模块要求，并以 AI 辅助模块争取额外加分。

## 系统架构

```
金价数据源 / 新闻数据源
       ↓
   爬虫模块 (requests + BS4)
       ↓
   SQLite 数据库 (金价时序 + 新闻)
       ↓
┌──────┴──────┐
│              │
数据分析       AI 辅助
(pandas +      ├── RAG 问答系统 (ChromaDB + DeepSeek API)
 matplotlib)   └── 深度学习探索 (ARIMA + LSTM)
       │
       ↓
   Streamlit 交互界面
```

## 模块设计

### 1. 爬虫模块

- **技术栈：** requests + BeautifulSoup
- **数据来源：** 东方财富/金十数据/新浪财经 公开页面及 API 接口
- **数据内容：**
  - 金价时序：日期、开盘、收盘、最高、最低、成交量
  - 黄金新闻：标题、发布时间、正文、来源
- **存储：** SQLite 数据库
- **设计原则：** 温和爬取（1-3s 间隔）、断点续爬、爬取与分析解耦

### 2. 数据分析与可视化

- **技术栈：** pandas + numpy + matplotlib + mplfinance
- **分析内容：** 基础统计、K 线走势、移动平均、波动率分析、周期性分析
- **图表产出：** K 线图、事件标注走势图、收益率分布直方图、月度对比图、预测对比图
- **输出格式：** PNG 图片供 PPT 使用

### 3. RAG 问答系统

- **技术栈：** ChromaDB（向量库）+ sentence-transformers（嵌入）+ DeepSeek API（LLM）
- **流程：** 用户提问 → 嵌入 → 检索相关新闻 → 拼入 prompt → LLM 生成回答
- **交互界面：** Streamlit Web 页面（命令行备选）
- **核心能力：** 针对黄金价格波动原因的自然语言问答

### 4. 深度学习预测（探索模块）

- **基线模型：** ARIMA（保证可运行）
- **探索模型：** LSTM 时序预测（效果待验证）
- **策略：** 独立子模块，不影响主体交付。有效则展示对比，无效则分析原因

## 关键技术决策

- 数据存储选择 SQLite 而非 MySQL/PostgreSQL：零配置、零部署
- 爬虫不采用 Scrapy：requests+BS4 学习成本低，大一队员可快速上手
- RAG 向量库选择 ChromaDB：轻量本地化，无需独立服务
- 交互界面选择 Streamlit：代码量极少，演示效果好
